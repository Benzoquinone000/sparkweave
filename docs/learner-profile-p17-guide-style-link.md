# 学习画像 P17：让导学明确引用推进风格

这一轮的重点，是把画像页里刚形成的“学习推进风格”，明确带回导学页。

## 问题

前面已经有两件事：

1. 画像页能给出“练习驱动型 / 概念澄清型 / 快速串联型 / 反复校准型 / 渐进压实型”
2. 导学页会根据画像动态调整资源入口和当前策略

但用户看到导学策略卡时，仍然可能会觉得：

> 你为什么现在要我先看图解，或者先做练习？

## 本轮目标

让导学策略卡把“因为你更偏这种学习方式，所以我现在这样安排”讲清楚。

## 本轮实现

在导学页的策略推断中，新增了一层轻量的“推进风格”判断，并把结果直接加入：

- `signals`
- `reasons`

也就是说，现在导学策略卡会明确显示：

- 推进风格：练习驱动型 / 概念澄清型 / 快速串联型 / 反复校准型 / 渐进压实型

同时会给一句对应解释，例如：

- 你更像是先通过练习暴露真实卡点，再用反馈压实理解
- 你更适合先把概念关系和边界看清楚，再进入练习
- 当基础够用时，你更适合先用短视频把流程串起来

## 价值

这一层的价值不在于新增更多模型能力，而在于把“个性化”从后台推断变成前台可感知体验：

- 用户更容易相信系统真的理解了自己
- 用户更容易接受“为什么现在先这样安排”
- 评委更容易看见画像如何直接驱动导学策略

## 下一步建议

下一步可以继续做两件事中的一件：

1. 把“推进风格”进一步回写到资源结果页，让用户知道这份图解/练习/视频是按什么风格生成的
2. 把“推进风格”做成可变化的长期趋势，展示学生经过多轮学习后风格是否变得更稳定
