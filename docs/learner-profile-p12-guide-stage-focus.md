# 学习画像 P12：导学阶段聚焦卡

这一轮继续优化的不是功能数量，而是导学页的使用阻力。

## 问题

虽然前面已经有：

- 学习闭环总览
- 当前任务区
- 反馈区
- 跳转和高亮

但对第一次使用的人来说，页面里还是容易出现一个心理问题：

> 我知道系统很完整，但我现在到底先点哪一个？

## 本轮目标

把导学页进一步压缩成“懒人式使用”：

- 当前只做哪一步
- 做完后会自动去哪里
- 如果想看全局，还有哪个入口

## 本轮实现

新增了一个轻量的“阶段聚焦卡”：

- `create` 阶段：
  - 先写学习目标
  - 做完后进入前测
- `diagnostic` 阶段：
  - 先做前测
  - 做完后回到当前任务
- `learn` 阶段：
  - 先完成当前任务
  - 完成后进入提交与反馈
- `feedback` 阶段：
  - 先看反馈
  - 再决定回到任务还是继续推进
- `complete` 阶段：
  - 先看总结和效果
  - 再决定保存或开启下一条路线

## 设计原则

这张卡故意保持轻量：

- 不重复大段系统解释
- 不和“学习闭环总览”抢主次
- 只回答“此刻先做什么”

这样用户不用读很多，就能继续往前走。

## 价值

这轮带来的价值主要有三点：

1. 降低导学页的第一次使用门槛
2. 让用户更少在多个模块之间来回试错
3. 让导学体验更接近“真实学习流程”而不是“功能面板”
