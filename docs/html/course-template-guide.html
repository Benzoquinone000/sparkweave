完整课程样例说明

赛题要求作品“自行构造至少一门完整高校专业课程”。SparkWeave 当前接入了两门课程模板：主线课程“深度学习”，以及新增的“智能机器人系统”。答辩和演示建议仍围绕深度学习展开，路径更稳；机器人课程用于说明系统可以迁移到另一类工程实践课。

这门课不是为了凑演示临时写的一组问题。它依据 ppts/深度学习/ 下的课件整理成 14 周、3 学分的高校专业课程，覆盖绪论、神经网络、CNN、深度学习软硬件、CNN 图像检索、多模态学习、RNN、注意力机制、Transformer、大模型应用、强化学习、无监督学习和深度生成模型。评委在学习页选择这门课后，可以看到课程目标、学习节点、任务、资源生成提示和学习报告闭环。

这门课适合做比赛主线：它本身就是高校 AI 方向常见课程，又天然需要图解、资料依据、练习反馈和分阶段学习路径。

机器人课程不是把旧课程换个名字，而是按新增课件重新组织了一条实践路线：先建立智能机器人系统全景，再进入运动机构、运动控制与规划、传感、导航定位，随后学习 ROS 基础、通信机制、roscpp/rospy、常用工具、TF/URDF，最后落到基于 ROS 的导航应用。评委如果想看第二门课程，可以在学习页选择“智能机器人系统”验证。

课程资料与 RAG

课程模板负责学习路线，课件资料负责回答依据。两者不能只做一边：如果只有模板，学习页能生成路径，但资料问答没有课程来源；如果只有资料库，学生又看不到按周推进的课程主线。

当前两门课程都已经约定了对应资料库：

课程 资料库 原始课件落点 当前状态
深度学习 深度学习 data/knowledge_bases/深度学习/raw/ 已同步 14 份课件，需按环境重建索引
智能机器人系统 智能机器人系统 data/knowledge_bases/智能机器人系统/raw/ 已同步 11 份课件，需按环境重建索引

新增课件后先同步原始资料：

python scripts/sync_course_materials_to_kb.py --stage-only

这一步只复制课件并更新资料库配置，不会假装已经完成向量化。资料问答要真正命中这些课件，还需要在 Docker Compose、Milvus 和 Embedding 服务可用时重建索引：

python scripts/sync_course_materials_to_kb.py --index

评委核验时可以先看资料页中是否出现 深度学习 和 智能机器人系统 两个资料库，再围绕课件内容提问。若资料库状态显示需要重新整理，点击页面里的重建索引或运行上面的 --index 命令即可。

考核方式

考核项 权重 关注点
章节小测 20% 神经网络、CNN、注意力、Transformer 等基础概念
实践练习 25% CNN 图像检索、模型对比或大模型应用链路
资料问答与证据记录 15% 回答能否追溯到课件或资料来源
课程项目报告 30% 模型路线、案例分析、错因复盘和后续计划
学习反思 10% 薄弱点、资源使用体验和下一步补救

这套考核方式支撑学习效果评估：不是只看一次答题分数，而是把练习、资料证据、反馈和反思放在一起看。

核验建议

核验时可以围绕这门课走一条稳定路线：

操作 说明
在学习页选择“深度学习” 确认系统内置一门完整高校专业课程
查看 DL1 到 DL14 的学习路线 确认课程有阶段、有依赖、有产出
做 D3 或 D5 查看 CNN 图解、图像检索流程和练习反馈
做 D8 或 D9 查看注意力机制、RAG 或 Transformer 图解
用资料页提问课件概念 查看资料来源和引用依据
回到记录页 查看练习反馈如何影响画像和下一步建议

这样走下来，看到的是 SparkWeave 如何支持一门真实专业课，而不是系统在讲自己。

核验方式

课程模板和课件入库可以直接运行校验：

python scripts/check_course_templates.py
python scripts/sync_course_materials_to_kb.py --stage-only

第一条命令检查课程结构、节点、任务、路径和必填字段。第二条命令检查模板里的课件路径能不能落到资料库。通过后，说明课程模板能被系统读取，资料也已经放到 RAG 的原始文件区；真实问答效果还需要重建索引，并在学习页、资料页、问问题页和记录页中走一遍。
