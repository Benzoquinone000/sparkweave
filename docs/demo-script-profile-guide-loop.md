# SparkWeave 演示脚本：画像驱动导学闭环

## 目标

给答辩、路演和录屏一个可以直接照着讲的脚本，重点突出：

1. 对话式学习画像自主构建
2. 个性化学习路径规划与资源推送
3. 多智能体协同生成资源
4. 学习效果评估与动态调整

## 推荐演示时长

- 5 分钟精简版
- 7 分钟比赛标准版

以下脚本以 7 分钟版本为准。

## 演示前准备

建议提前准备一条稳定主题，例如：

- 高等数学：梯度下降 / 极限 / 导数应用
- 机器学习基础：损失函数、梯度下降、过拟合
- C++ / ROS 入门：发布订阅、节点通信、调试流程

建议准备状态：

- `/memory` 里已经有初始画像
- `/guide` 至少有一条可以演示的路线
- 当前路线里已有图解、练习或短视频中的至少一种

## 讲述主线

### 1. 开场：系统不是工具箱，而是学习闭环

可以直接说：

> SparkWeave 不是把多个 AI 工具堆在一起，而是围绕学习者画像，把导学、资源生成、练习反馈和效果评估串成一个持续更新的闭环。

### 2. 从画像页开始

进入 `/memory`，重点展示：

- 当前画像概览
- 系统最建议现在处理的一件事
- 最近画像变化和“证据结论”
- 用户可以校准的判断
- 进入导学的入口

讲解重点：

> 这里不是静态标签，而是系统根据对话、导学、练习、反思、题目本和笔记持续汇总出的学习者模型。

切到“依据”页时，不要逐条读证据。先讲顶部“证据结论”卡片：它会把最近行为、累计证据量、资源类型和来源压缩成用户能理解的一句话。评委如果追问“画像从哪来”，再展开下面的证据列表。

建议指出：

- 当前重点是什么
- 当前薄弱点是什么
- 为什么系统这么判断
- 下一步建议是什么

### 3. 从画像跳到导学

点击“进入导学”。

展示点：

- 导学会带入画像中的目标、薄弱点和学习风格
- 首屏只保留当前任务和下一步动作
- 复杂内容进入子页或任务产物区，不在首屏平铺

讲解重点：

> 画像不会停留在展示层，而是直接驱动导学路线创建。用户不用重新描述一遍问题，系统已经把关键上下文接住了。

### 4. 从 Chat 展示“按画像继续”

切到 `/chat`，不要手动输入复杂提示词，直接点击首屏的“按画像继续”。

展示点：

- 前端只发送“继续学习”，用户不需要理解内部能力配置
- 对话协调智能体会读取画像，把泛化指令改写成具体学习任务
- 协作面板显示“画像触发”和“接力路线”，说明画像如何唤醒题目、图解、动画或公开视频智能体

讲解重点：

> 这是懒人式入口：学生只表达“我想继续”，系统自己决定下一步该补概念、做练习、看图解还是找视频，并把多智能体接力过程压缩成用户能看懂的一句话。

如果时间紧，这一步只停留 20 秒：点“按画像继续”，展示“画像触发”的改写结果和横向接力路线即可，不展开原始事件明细。

### 5. 展示懒人式当前任务

进入 `/guide` 后，重点展示“先做这一件事”。

依次讲三点：

1. 现在要补什么或学什么
2. 为什么先做它
3. 做完后去哪里提交反馈

讲解重点：

> 导学不是把所有工具摆给学生，而是替学生做选择：现在只做最该做的一件事。

可强调：

- 页面减少下拉和大段面板
- 资源、练习、提交分到明确页面或子框
- 用户不需要理解背后的工具，只需要沿着按钮往前走

### 6. 展示资源生成不是随机的

在当前任务资源区展示：

- 当前任务需要的图解 / 练习 / 短视频
- 参考了哪些画像信号
- 资源卡里的“生成依据”

讲解重点：

> 资源不是让学生自己找，而是系统围绕当前任务生成。生成依据来自画像、任务状态和刚刚的学习反馈。

### 7. 展示资源生成结果

进入生成结果区，展示任一资源：

- 图解
- 交互练习
- 短视频

重点展示两个轻量信息：

- “生成依据”：参考了哪些画像信号、针对了哪些薄弱点
- “协作链路”：画像智能体、图解/动画/出题智能体、评估智能体如何接力
- Chat 里的“接力路线”：把对话协调、检索、生成、校验、讲解等角色压缩成一条横向路线

讲解重点：

> 这里能看见多智能体协同，但默认只展示学习者需要理解的摘要，不展示运行日志和原始 JSON。

如果展示精选公开视频，点击“打开观看”后停一下，指出卡片出现“已记入画像依据”。这能证明网络资源不是一次性推荐，而是会回到学习画像，影响后续资源偏好。

### 8. 提交反馈，展示回写

进入提交页，提交：

- 掌握评分
- 一句话反思

然后回到反馈区或画像页，展示：

- 学习反馈
- 画像闭环提示
- 画像刷新后最近变化
- 学习报告里的“学习处方”：系统把分数、错因和画像信号压缩成一个首要行动，并可以直接生成图解、练习、复测或进入课程产出包
- “演示就绪”：确认画像、资源、练习、报告和作品证据已经成链，缺口会给出一个最小补齐动作

讲解重点：

> 学生每一次学习行为都会回写系统，系统再调整下一步推荐，这就形成了真正的学习闭环。

### 9. 收尾：对应赛题要求

结尾建议直接对照赛题：

- 对话式学习画像：通过画像页、校准、证据链完成
- 多智能体资源生成：通过图解、短视频、练习等能力协同完成
- 个性化导学规划：通过导学页任务拆解和动态推荐完成
- 智能辅导：通过图解、问答、练习反馈、多模态资源完成
- 学习效果评估：通过学习效果评估、反馈回写和下一步建议完成

课程产出包里的“7 分钟演示路线”可以作为录屏提词器：只展示关键片段，完整赛题映射和兜底材料保存在 Markdown 中。

录屏前先看“录屏兜底包”：确认固定学习者画像、已保存产物、可现场生成素材和兜底话术。这样即使现场视频渲染或模型响应变慢，演示也不会断线。

推荐直接使用“机器学习基础稳定演示样例”：按 T1 建立全景图、T4 生成梯度下降图解、T6 做模型评估练习的顺序推进。它覆盖画像、路径、资源、练习、反馈和报告，最适合 7 分钟录屏。

如果从空白状态开始录屏，进入 `/guide` 后优先点击“开始稳定演示”。系统会自动创建机器学习课程 Demo 路线，省去手动填写目标和选择模板的时间。

进入 T4 或 T6 这类稳定样例任务时，优先点击任务页里的“使用稳定提示词生成”。提交任务时可以点击“示例反馈”的“带入”，让分数和反思保持一致，方便录屏复现。

稳定 Demo 主流程顶部会出现“录屏下一步”。录屏时跟着这个小提示走即可：它只保留当前按钮和一句讲解，不要展开其它面板抢节奏。

提交反馈后，停留在反馈区的“演示收尾”卡片：先讲一句“学习证据已经回写画像”，再点“看路线”说明路径如何变化，最后点“看产出包”展示 7 分钟演示路线和兜底材料。

打开课程产出包后，优先讲顶部的“录屏检查”卡片：一句话说明就绪状态，按前三个片段走脚本，最后补一句兜底动作。其它评分标准和复习计划可以只作为提交材料，不必在 7 分钟里展开。

如果后端兜底数据来自英文模板，前端会把录屏检查、课程产出包和下一步动作转换成中文表达；录屏时只讲用户能理解的学习路线，不展示工程占位文案。

## 演示时尽量避免

- 直接展示太多调试信息
- 在一页里停留过久讲字段
- 先讲技术栈再讲用户价值
- 同时展开太多侧边区或历史内容

## 一句话版本

> SparkWeave 先理解学生，再安排学习，再生成资源，再根据反馈持续调整。
