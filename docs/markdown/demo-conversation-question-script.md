# 真实会话逐问展示台词稿

这份稿子根据当前录屏会话写，每个用户问题对应一段可直接讲的台词。录制时可以按页面里已经生成的结果逐条往下讲，不需要重新编排功能顺序。

核心讲法：这不是在展示“我问一句，模型答一句”，而是在展示 SparkWeave 如何根据学习记录自动选择不同智能体：研究规划、精选视频、知识可视化、普通讲解、资料库 RAG、动画生成。每一轮都要点出“为什么系统这样调度”和“结果如何回到学习记录”。

## 问题一：我想入门深度学习，为我梳理一下知识点

页面画面：停在第一轮回答，展示“深度研究”“Research Outline”“处理路线 7 项”。

台词：

这一轮我输入的是一个很宽泛的学习目标：我想入门深度学习，让系统梳理知识点。

可以看到，系统没有直接给一段零散回答，而是自动进入了“深度研究”流程。它先把原始问题改写成更适合学习规划的任务：覆盖神经网络基础、CNN、RNN、注意力机制、Transformer、大模型应用、无监督学习和生成模型，并要求给出学习顺序和阶段目标。

这里的结果被整理成三个阶段：第一阶段打基础，理解前向传播、反向传播、梯度下降和多层感知机；第二阶段学习 CNN、RNN、注意力机制和 Transformer；第三阶段进入大模型应用、无监督学习和生成模型。

这一轮要展示的技术点是自动调度。用户只是说“我想入门”，系统会结合学习记录判断这不是单点问答，而是一个学习路径规划问题，所以交给研究和学习路径流程。页面里的“理解任务、拆解主题、查找资料、整理报告”就是智能体接力的过程。

可以补一句：

这里已经开始体现个性化。系统不是只根据当前一句话回答，还会结合学习记录里的薄弱点、偏好、时间预算和下一步任务，把大问题变成可执行的阶段目标。

## 问题二：为我找几个入门视频

页面画面：展示“精选视频 3 个推荐”、当前重点、优先照顾、偏好、水平，以及“我看了这个，记入学习记录”按钮。

台词：

第二轮我没有重新解释背景，只说“为我找几个入门视频”。系统能接住上一轮的学习上下文，知道我现在是深度学习初学者，当前重点是神经网络基础、CNN、RNN、注意力机制和 Transformer。

这里系统自动进入“精选视频”流程，返回 3 个公开视频推荐。它不是随便搜几个结果，而是会结合当前画像：当前重点、优先卡点、学习偏好和 beginner 水平。可以看到页面里明确写着，它会先照顾“人工智能与深度学习绪论、前馈神经网络”这些当前更容易卡住的点。

这一轮的技术创新点是资源推荐不是孤立搜索。外部视频搜索结果会经过排序筛选，再和学习画像对齐。学生看完后可以点击“我看了这个，记入学习记录”，这个行为会回到学习证据里，让后续推荐更准。

录制时可以这样强调：

普通搜索引擎只给链接，SparkWeave 会告诉学生先看哪个、怎么看、看完后如何反馈。它把外部资源变成学习路径的一部分。

## 问题三：图解神经网络训练过程

页面画面：展示“知识可视化”、Mermaid flowchart、训练流程图，以及处理路线里的“分析结构、设计图解、生成图解、Reviewing/validating/optimizing code”。

台词：

第三轮我输入“图解神经网络训练过程”。系统判断这是一个可视化学习资源生成任务，所以自动进入“知识可视化”流程。

页面上这张图把一次神经网络训练迭代讲清楚了：准备数据、初始化模型、前向传播、计算损失、反向传播、优化器更新参数，再判断是否达到停止条件，最后进入验证和测试。

这里要重点讲技术实现：系统不是只生成一段说明文字，而是生成可渲染的 Mermaid 图解代码。它会先分析结构，再设计图解，再生成代码和预览结果。后面还有一步 Review、validation 和 optimization，用来检查 Mermaid 语法、标签措辞和渲染兼容性。

可以指着页面里的“检查建议”说：

这里显示系统已经检查过 Mermaid flowchart 语法，并优化了标签和连接写法。这说明图解生成不是一次性输出，而是带有代码校验和自修复意识的资源生成流程。

技术创新点补一句：

这就是我们说的“可渲染资源生成”。图解不是模型口头描述一张图，而是生成结构化源码、预览和检查结果，后续可以保存到学习记录中复用。

## 问题四：我现在想学习 DPO，你知道它的公式吗

页面画面：展示 DPO 公式解释回答，以及处理路线里“Keeping this turn in the default chat”。

台词：

第四轮我问的是“我现在想学习 DPO，你知道它的公式吗”。这一次系统没有强行进入复杂智能体，而是保持在默认讲解流程里。

这其实很重要。多智能体系统不是每个问题都要调一个重流程。对于这种概念解释问题，默认对话就足够。系统会把 DPO 的数据形式、核心损失函数、当前模型、参考模型、winner、loser、beta 这些项逐步解释清楚。

页面里可以看到，它把 DPO 讲成“让模型更偏向人类喜欢的回答，同时不要偏离参考模型太远”。这适合初学者理解，也符合前面学习画像里 beginner 的状态。

这一轮的技术点是能力路由的克制。系统不是为了展示智能体而展示智能体，而是根据问题复杂度决定是否委派。能直接解释的就直接解释，需要检索、图解、练习或动画时再交给专业能力。

可以补一句：

这种“该调度时调度，不该调度时保留默认对话”的设计，能减少不必要的延迟和成本，也让学生的学习体验更自然。

## 问题五：资料库中都介绍了哪些注意力机制

页面画面：展示注意力机制表格、资料来源、5 条来源、检索过程和第 84 页来源片段。重点展开“查看来源与检索过程”，让评委看到 Agentic RAG 的中间过程。

台词：

第五轮我问“资料库中都介绍了哪些注意力机制”。这次问题明确要求从课程资料里找内容，所以系统进入 Agentic RAG 流程。

回答里列出了自注意力、多头自注意力、空间注意力、跨模态注意力、稀疏注意力、层次化注意力、门控注意力等机制，并给出适合初学者的学习顺序。

这一轮最重要的不是答案本身，而是答案是怎么来的。可以看到页面下面有“来源可用”“查看来源与检索过程”。系统没有只把问题丢给大模型，而是先判断这个问题是否需要拆分：因为“资料库中都介绍了哪些注意力机制”比较集中，所以本轮不额外拆子问题，走 fast 检索策略；如果问题更复杂，比如要求比较不同注意力机制的应用场景，系统就会生成 query plan，把问题拆成多个检索方向。

接下来系统从“深度学习”资料库中召回来源片段，并做可用性判断。这里找到了深度学习课程第 9 章“注意力机制与外部记忆”的课件，尤其第 84 页总结里直接列出了注意力机制的类型。页面还展示了来源页码、相关度和命中关键词。也就是说，回答里的机制列表不是模型凭记忆编出来的，而是被课程资料支撑的。

这就是我们 Agentic RAG 的设计重点：第一步先理解问题，判断是否需要 query plan；第二步选择检索策略，比如 fast、exact、broad 等；第三步召回资料并组织来源；第四步评估证据是否足够；最后才生成面向学生的回答。前端把这些中间过程展示出来，评委可以直接展开核验。

技术创新点补一句：

这部分是 SparkWeave 很重要的 Agentic RAG 设计。它不是“检索几段文本塞给模型”这么简单，而是把问题理解、检索规划、策略选择、证据召回、质量判断和来源展示做成一个可观测流程。简单问题可以快速检索，复杂问题可以拆成多路查询；证据足够时生成回答，证据不足时进入修复或保守回答。这样资料问答既能利用大模型组织语言，又能保留课程证据链，降低幻觉风险。

## 问题六：动画讲解 DPO

页面画面：展示“数学动画”“处理中”、处理路线“理解任务、分析概念、设计分镜、生成结果”，以及 learning_goal 和 storyboard 进度。

台词：

最后一轮我输入“动画讲解 DPO”。这说明学生在看完公式解释后，希望换一种更直观的方式理解。系统识别到这是动画生成任务，于是进入“数学动画”流程。

页面里可以看到，它先分析学习目标，把 DPO 的核心直觉改写成动画目标：用偏好数据，让模型更倾向于人类喜欢的回答，同时避免偏离原模型。接下来进入设计分镜，也就是把一个抽象公式拆成画面步骤。

这部分要重点讲动画生成的技术链路。SparkWeave 的动画不是固定模板，而是智能体根据学习目标生成分镜，再生成可执行动画代码，通常走 Manim 风格的渲染流程。如果代码报错，系统会根据错误反馈重试；如果渲染结果异常，还可以进一步检查画面是否空白或错位。

录制时如果动画还在生成，不要干等，可以这样说：

动画生成会比普通回答慢，因为它不是只输出文字，而是要经历概念分析、分镜设计、代码生成、渲染和错误修复。现在页面上展示的“分析概念”和“设计分镜”，就是这个多智能体生成流程的中间状态。

技术创新点补一句：

这一轮展示的是代码驱动的多模态资源生成。系统把抽象公式讲解变成可视化脚本和动画结果，让 DPO 这种偏理论的内容可以用动态方式学习。

## 串场总台词

如果你想在几轮问题之间自然过渡，可以这样讲：

刚才这几轮其实展示的是同一条学习链路。

第一轮，“我想入门深度学习”，系统进入深度研究，帮我规划学习阶段。

第二轮，“找几个入门视频”，系统根据刚才形成的学习重点和画像，推荐外部视频资源，并允许我把观看行为写回学习记录。

第三轮，“图解神经网络训练过程”，系统进入知识可视化，生成可渲染的 Mermaid 图解，并做代码检查。

第四轮，“学习 DPO 公式”，系统没有过度调度，而是保持默认讲解，把公式拆成初学者能读懂的结构。

第五轮，“资料库中有哪些注意力机制”，系统走 RAG，从深度学习课件里检索并展示来源。

第六轮，“动画讲解 DPO”，系统进入数学动画，把抽象公式转成分镜和动画生成任务。

这说明 SparkWeave 的多智能体不是摆一排按钮，而是根据学生问题自动选择合适能力。研究、视频、图解、RAG、公式解释和动画生成都围绕同一个学习目标展开，最后又能通过保存、反馈和学习记录回到画像里。

## 最后总结台词

这段真实会话能体现 SparkWeave 的几个技术创新点。

第一是自动调度。系统会根据问题类型选择深度研究、精选视频、知识可视化、默认讲解、RAG 或数学动画，而不是让学生理解复杂的工具菜单。

第二是 Agentic RAG。资料库问题会展示来源、页码、相关度和检索过程，回答可以被核验。

第三是多模态资源生成。图解生成会产出 Mermaid 代码并校验，动画生成会经历分镜、代码、渲染和错误修复。

第四是学习画像闭环。视频观看、图解有帮助、保存记录、练习结果这些行为都可以写回学习记录，影响后续推荐。

所以这不是一次普通聊天演示，而是一段从学习目标出发，连续调用多智能体能力，并把结果沉淀回学习系统的端到端学习过程。
