学习画像与记忆设计

SparkWeave 的学习画像不是让学生先填一张很长的表，而是在真实学习过程中慢慢形成：学生说出目标，完成导学任务，做题，保存笔记，修正系统判断，这些都会变成可追溯的学习记录。下一次系统给出学习建议时，会先看这些记录，再决定先补哪个点、用什么资源、安排多长时间。

这部分对应赛题里的“对话式学习画像自主构建”和“学习效果评估”。评委可以在 记录与画像页 看到它的前端呈现。

页面上能看到什么

记录页默认不展示复杂数据表，而是先回答学生最关心的一句话：现在该做什么。

页面区域 含义
概览 展示当前学习重点、优先卡点、下一步行动和建议稳定度
记录来源 展示系统参考过哪些导学、练习、笔记和修正记录
补充 允许学生手动补充长期目标、背景或偏好
判断准确 学生确认系统判断，让画像更稳定
不准？一句话改掉 学生直接修正画像，例如“我卡在公式含义，不是基础概念”

这样做的目的很简单：画像不是藏在后台的标签，而是学生能看见、能质疑、能改正的学习依据。

画像记录哪些内容

画像快照主要分成四类信息：

信息 例子 用途
学习目标 “想补梯度下降的直观理解” 决定导学路线和下一步任务
资源偏好 图解、练习、短视频、代码示例 影响资源生成和推荐形式
薄弱点 概念边界不清、推导步骤不稳、题型易错 决定优先补哪里
掌握状态 某个知识点已掌握、练习中、需要补基 影响复习、练习和进阶安排

这些信息不是只靠一次问答生成。系统会把多种来源合在一起，并保留每条判断来自哪里。

记录从哪里来

来源 系统怎么使用
对话与导学 学生在导学页说出的目标、卡点、时间预算和偏好会进入画像
导学任务反馈 任务完成情况、反思、得分和错因会更新薄弱点和掌握状态
题目本 练习作答、正确率、题型和知识点用于判断掌握度
学习笔记 学生保存的资料、图解、问答和资源会作为学习兴趣与资源偏好的参考
手动修正 学生确认、否定或改写系统判断后，会形成一条更高可信度的记录
语音与多模态练习 语音输入、口语练习或图片题结果也可以作为学习证据进入记录

正式记录保存在追加式文件 data/user/learner_profile/evidence.jsonl 中；画像快照保存在 data/user/learner_profile/profile.json 中。这样评委既可以从页面看结果，也可以从文件看到记录确实落地。

证据账本到画像快照

画像链路分三层，不把所有学习记录直接塞给大模型。

层次 代码位置 做什么
证据账本 sparkweave/services/learner_evidence.py 用追加式 JSONL 保存标准化事件，字段包括来源、动作、对象、课程、节点、任务、资源类型、得分、错因、置信度和元数据
画像聚合 sparkweave/services/learner_profile.py 从长期记忆、导学会话、题目本、笔记和证据账本里提取目标、偏好、薄弱点、掌握状态和下一步行动
上下文注入 sparkweave/services/profile_context.py、sparkweave/runtime/context_enrichment.py 只提炼 current_focus、weak_points、preferences、mastery_needs_attention、next_action 等短提示，和会话上下文一起交给智能体

这个分层能避免两个问题：一是模型看到过多原始记录导致回答不稳定；二是画像变成不可解释的“黑箱标签”。评委如果想核验，可以先看记录页的“记录来源”，再打开 evidence.jsonl 和 profile.json，两边能对得上。

一次画像更新怎么发生

以学生完成一次导学练习为例：

步骤 发生了什么 后续影响
1. 学生提交答案 系统记录题目、作答、得分、知识点和错因 形成一条学习证据
2. 更新导学会话 当前任务状态、掌握度和下一步建议刷新 学习页能继续推进
3. 写入记录 练习结果进入统一证据记录 记录页能看到来源
4. 刷新画像 聚合目标、偏好、薄弱点和掌握状态 下一次问答会更贴合学生
5. 生成下一步行动 可能推荐补基图解、诊断题、复测或继续学习 学生回到可执行任务

如果学生觉得判断不准，可以在记录页直接修正。系统会把这次修正当作明确反馈，而不是继续沿用旧判断。

代码里这一步不是前端临时改文案。记录页点击“判断准确”或提交一句话修正后，会调用 POST /api/v1/learner-profile/calibrations。后端用 build_profile_calibration_event() 写入一条 profile_calibration 事件，再刷新画像。被学生否定的判断会进入 rejected claims，被修正的判断以更高置信度参与下一次聚合。

学习效果如何参与推荐

学习效果评估不会只看“最近一次回答好不好”。它会综合最近的学习证据，给出几个更接近学习现场的判断：

判断 说明
掌握度 某个概念是否已掌握、正在练习或需要补基
错因状态 最近是否还有未处理的错误信号
资源行为 学生是否看过图解、视频、资料或保存过笔记
趋势 最近练习表现是变好、持平还是下降
下一步动作 建议诊断、补基、练习、复测、复习或进阶

这些结果会进入学习页和问问题页。比如学生常在“极限定义”上答错，系统下一次不会急着推新章节，而是更可能安排一组低门槛练习，或者先给一张概念边界图。

对问答和资源生成的影响

画像会以简短提示进入后端运行上下文，帮助系统调整回答方式。它不会把完整记录原样塞给模型，而是提炼成几类提示：

提示 例子 影响
当前重点 正在学习“梯度下降” 回答围绕当前课程展开
薄弱点 “不理解学习率为什么会影响收敛” 解释会先处理卡点
偏好资源 更喜欢图解和练习 优先生成图解、题目或短任务
时间预算 本轮只有 10 分钟 建议会控制任务大小
下一步建议 先做诊断题或补基图解 问答页能直接接上学习路径

相关能力包括普通问答、资料问答、图解生成、数学动画、练习生成和导学路线。前端会用“已结合记录”“按你情况调整”等轻量提示说明系统确实使用了学习画像。

接口 作用

演示时可以这样看

  1. 在学习页创建或继续一条导学路线，完成一次练习或反馈。
  2. 进入记录页，观察当前重点、优先卡点和下一步行动是否变化。
  3. 打开记录来源，查看建议参考了哪些导学、练习或笔记记录。
  4. 在概览里点击“判断准确”或用一句话修正画像。
  5. 回到学习页或问问题页，看系统是否按新的画像继续安排任务。

这条演示线能体现“学生行为进入画像，画像影响资源与路径，新的学习结果再回到记录”。

边界说明

  • 画像依赖真实学习记录。刚启动系统时，如果记录很少，页面会引导学生先完成导学或练习。
  • 系统会显示建议稳定度，避免把证据不足的判断说成确定结论。
  • 手动补充和修正会进入正式记录，后续建议会参考学生自己的修正。
  • 画像用于学习推荐和资源生成，不用于课程外的用户评价。
  • 运行数据位于 data/user/ 下，提交源码时可按交付要求保留样例数据，避免混入个人隐私记录。
