# AI 助教中心 SparkBot 演示 Runbook

这份 Runbook 面向 `/agents` 的 AI 助教中心，用于补齐计划书 P4 的“固定演示助教模板、一键 seed、7 分钟录屏脚本和 AI Coding 说明”。

## 一键准备

```powershell
python -m sparkweave_cli sparkbot seed-demo
sparkweave serve --port 8001
cd web
npm run dev
```

打开 `http://127.0.0.1:3782/agents`，选择「大模型与智能学习系统助教」。

如需覆盖旧的演示工作区：

```powershell
python -m sparkweave_cli sparkbot seed-demo --overwrite
```

如果希望 seed 后直接启动助教：

```powershell
python -m sparkweave_cli sparkbot seed-demo --start
```

## Seed 内容

默认助教 ID：`ai_learning_agents_systems_tutor`

工作区会写入：

- `SOUL.md`：课程助教人格，强调画像、资料、资源生成和学习效果闭环。
- `USER.md`：固定演示学习者画像。
- `TOOLS.md`：讯飞星火、OCR、语音听写、TTS、RAG 和题目生成能力说明。
- `AGENTS.md`：画像、路径、检索、讲解、出题、评估的协作路线。
- `HEARTBEAT.md`：录屏前检查与主动提醒。
- `NOTES.md`：稳定提示词和演示目标。
- `COURSE.md`：完整课程定位、目标、项目产物和考核结构。
- `LESSONS.md`：8 周课程安排、课堂活动和录屏提示词。
- `QUESTION_BANK.md`：概念题、判断题、实践任务和参考答案。
- `RUBRIC.md`：课程项目评分标准和赛题评分映射。
- `RESOURCES.md`：页面入口、讯飞工具链讲法和演示兜底。

默认不覆盖已有用户编辑；使用 `--overwrite` 才会重写这些文件。

打开 `/agents` 后进入「资料与产物」区域，文件列表会优先显示 `COURSE.md`、`LESSONS.md`、`QUESTION_BANK.md`、`RUBRIC.md`、`RESOURCES.md` 这些课程资料。录屏时先展示课程资料包，再切到 `SOUL.md` 或 `AGENTS.md` 说明长期助教和多智能体协作。

同一区域会显示「比赛演示检查」，用于录屏前核对完整高校课程、学习画像与路径推荐、课程资料智能辅导、多智能体资源生成、讯飞多模态工具和学习效果评估闭环。右侧的 7 分钟路线和提交包导出命令可直接作为现场提示。

## 7 分钟路线

| 时间 | 页面动作 | 讲法 |
| --- | --- | --- |
| 0:00-0:40 | 打开 AI 助教中心，展示今日建议 | 学生进来先看到“今天做什么”和推荐依据 |
| 0:40-1:30 | 点“开始今天的学习” | 助教读取学习画像、最近练习和课程资料 |
| 1:30-2:30 | 在对话里提问 RAG 证据链 | 展示来源约束答疑和学习语言状态 |
| 2:30-3:10 | 打开“资料与产物”，展示学习协作路线 | 画像、资料、讲解、练习、多模态和评估回写形成一条闭环 |
| 3:10-4:00 | 点“生成图解”或“讯飞语音脚本” | 同一问题可转成图解结构和 TTS 讲解音频 |
| 4:20-5:20 | 点“生成练习”或学习效果 next action | 展示多智能体出题和个性化复测 |
| 5:20-6:20 | 对回答点“有帮助/太难了/想要例子” | 反馈写入学习效果事件，形成评估闭环 |
| 6:20-7:00 | 展示 `COURSE.md`、`QUESTION_BANK.md` 和产物卡 | 收束到完整课程资料、PPT/视频材料和 AI Coding 说明 |

## 录屏提示词

可直接从 `NOTES.md` 复制：

- 请基于我的学习画像和课程资料，告诉我今天最应该完成的一步。
- 请把 RAG 证据链用图解方式讲清楚。
- 请生成 3 道多智能体协作机制小测，并等我作答后分析错因。
- 请生成一段适合科大讯飞 TTS 合成的 60 秒语音讲解脚本。
- 请复盘我刚才的回答，把错因和下一步写入学习记录。

## 赛题映射

| 赛题要求 | AI 助教中心展示点 |
| --- | --- |
| 对话式学习画像自主构建 | 首屏今日建议、推荐原因、反馈写入学习证据 |
| 多智能体协同资源生成 | 页面“学习协作路线”、`AGENTS.md` 协作路线、讲解卡、练习卡、图解卡 |
| 个性化学习路径规划和资源推送 | 学习效果 next action、开始今天的学习、完成后记录 |
| 智能辅导 | 课程资料答疑、图解、OCR 讲义识别提示、TTS 语音脚本 |
| 学习效果评估 | 回答反馈、完成动作、错因复盘、下一步动态调整 |
| 科大讯飞工具 | `TOOLS.md` 和多模态资源动作中明确出现星火、OCR、语音听写、TTS |

## AI Coding 说明

提交材料引用 `docs/ai-coding-statement.md`。演示时只需要一句话：

> 本项目使用 AI Coding 辅助规划、代码实现、测试和文档整理，但所有功能取舍、运行验证、密钥边界和提交材料均由项目维护者人工复核。

最终完成证据可直接看 `docs/sparkbot-plan-completion-report.md`，里面已经把 P0-P4 的完成矩阵和验证命令整理好了。

## 兜底

- 助教未在线：点击首屏“启动助教”或运行 `python -m sparkweave_cli sparkbot seed-demo --start`。
- 模型响应慢：打开“资料与产物”，展示 seed 的 `NOTES.md` 稳定提示词和产物卡。
- 讯飞服务暂不可用：展示脚本和接入位置，说明系统支持 provider 降级，不阻断文字助教闭环。
- 时间不够：只演示“今日建议 -> 图解/TTS 脚本 -> 回答反馈写入学习证据”三步。
