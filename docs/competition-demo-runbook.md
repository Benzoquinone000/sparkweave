# 比赛 7 分钟演示 Runbook

这份 Runbook 用于正式录制或答辩前排练。目标不是展示所有功能，而是在 7 分钟内把赛题五条要求讲成一条清楚的学习闭环。

推荐主路线：`/guide` -> “开始稳定演示” -> 当前任务 -> 资源生成 -> 提交反馈 -> 学习报告 -> 课程产出包。

## 录制前检查

先完成这 6 件事：

1. 后端和前端都已启动，浏览器打开 `http://localhost:3782`。
2. `/guide` 能进入“开始稳定演示”。
3. Manim、FFmpeg、LaTeX 已配置；如果视频慢，就只展示历史产物或脚本结构。
4. 设置页至少有一个可用问答模型。
5. 课程产出包能打开，并能看到“录屏检查”“赛题五项对齐”“答辩问答预案”“比赛提交清单”。
6. 点一下“下载 Markdown”，确认能导出课程材料。

可选命令：

```powershell
python scripts\check_course_templates.py
pytest tests\services\test_guide_v2.py -q
cd web
npm run lint
npm run build
npx playwright test tests/e2e/guide-v2-demo.spec.ts --project=desktop
```

## 7 分钟主线

### 0:00-0:40 项目价值

页面：`/guide`

动作：

- 点击“开始稳定演示”。
- 等待路线创建完成。

讲法：

> SparkWeave 不是普通聊天机器人，而是先理解学习者，再规划路线，再生成资源，最后根据练习和反思调整下一步。

对应赛题：

- 对话式学习画像自主构建
- 个性化学习路径规划

### 0:40-1:40 当前任务

页面：导学当前任务

动作：

- 展示“先做这一件事”。
- 不展开太多面板，只讲当前任务、为什么先做它、做完去哪提交。

讲法：

> 用户不需要在一堆工具里找入口，系统把下一步压成一个明确任务。

对应赛题：

- 个性化学习路径规划和资源推送

### 1:40-3:10 多智能体资源生成

页面：当前任务资源区

动作：

- 优先点击稳定提示词生成图解。
- 如果现场慢，展示录屏检查里的稳定兜底素材。
- 展示资源卡的“生成依据”和“智能体接力”。

讲法：

> 画像智能体判断卡点，协调智能体选择能力，图解/出题/动画/检索智能体生成资源，评估智能体把结果带回学习闭环。

对应赛题：

- 多智能体协同的资源生成
- 智能辅导

### 3:10-4:30 交互练习与反馈

页面：提交页

动作：

- 点击“进入提交页”。
- 使用示例反馈带入。
- 提交评分和反思。

讲法：

> 练习不是展示题目而已，提交后会形成学习证据，回写画像和学习报告。

对应赛题：

- 智能辅导
- 学习效果评估

### 4:30-5:40 学习报告

页面：反馈区 / 学习报告

动作：

- 展示“演示收尾”。
- 打开学习报告或路线变化。
- 重点讲学习处方，不读所有指标。

讲法：

> 系统把得分、错因、画像信号压缩成下一步行动，而不是只给一个分数。

对应赛题：

- 学习效果评估
- 动态调整学习资源推送策略

### 5:40-7:00 课程产出包收尾

页面：课程产出包

动作：

- 打开“看产出包”。
- 展示“录屏检查”“赛题五项对齐”“答辩问答预案”“比赛提交清单”。
- 点击或指出“下载 Markdown”。

讲法：

> 这里把学习过程整理成可提交材料：演示路线、PPT 骨架、录屏讲稿、赛题映射、答辩问答和 AI Coding 说明。

对应赛题：

- 配套文档
- 演示 PPT
- 智能体演示视频
- 可完整运行项目材料

## 现场兜底

| 问题 | 处理 |
| --- | --- |
| 模型响应慢 | 使用稳定提示词或课程产出包里的兜底素材说明 |
| Manim 渲染慢 | 展示图解、Manim 脚本结构或历史视频截图 |
| 搜索/RAG 不稳定 | 只演示画像、导学、资源、练习、报告这条本地闭环 |
| 时间不够 | 跳过 Chat，直接从 `/guide` 的稳定演示入口开始 |
| 页面信息太多 | 只停留“当前任务”“提交反馈”“课程产出包”三个页面 |

## 答辩时优先回答

1. 为什么不是普通聊天机器人？
   SparkWeave 有画像、路线、资源、反馈和评估闭环。

2. 多智能体在哪里？
   资源卡和 Chat 协作路线会展示画像、协调、图解/动画/出题/检索、评估的接力。

3. 画像怎么更新？
   对话、前测、练习、反思、Notebook 保存和资源使用都会形成证据；用户也可以确认或修正画像。

4. 学习效果怎么评估？
   系统综合任务完成、练习得分、错因、反思、资源使用和画像趋势，输出学习处方。

5. AI Coding 如何说明？
   课程产出包和 `docs/ai-coding-statement.md` 写明辅助范围、人工复核、密钥边界和可追溯材料。

## 录制后整理

录完后建议导出：

- 课程产出包 Markdown。
- 演示视频。
- PPT 大纲截图或整理版 PPT。
- 项目 README。
- `.env.example` 和依赖清单。
- `docs/ai-coding-statement.md`。

最终讲述核心保持一句话：

> SparkWeave 先理解学生，再安排学习，再生成资源，再根据反馈持续调整。
