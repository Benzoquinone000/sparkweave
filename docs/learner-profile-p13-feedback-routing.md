# 学习画像 P13：反馈后的清晰分流

这一轮继续优化导学页里的“反馈阶段”。

## 问题

之前的即时学习反馈已经能展示：

- 分数
- 总结
- 知识点反馈
- 资源动作
- 画像回写

但它更像一张“结果卡”，还不够像一张“下一步决策卡”。

对于普通用户来说，最真实的问题是：

> 我看到反馈了，但我现在到底该继续、该补救，还是该复测？

## 本轮目标

把反馈阶段从“展示结果”改成“直接分流”：

- 高分时，明确告诉用户可以继续推进
- 中等分时，明确告诉用户先稳一下再走
- 低分时，明确告诉用户先补救再复测

## 本轮实现

### 1. 反馈卡新增“现在建议你这样走”

新的反馈分流区会根据：

- `score_percent`
- `resource_actions`
- `next_task_title`

自动生成最多 3 条可执行路径。

### 2. 三类路径更直白

当前会自动组织出这些方向：

- 继续推进
- 先补救
- 再做复测
- 顺手巩固
- 回到当前任务
- 进入完整路线页

### 3. 直接动作，不再只给总结

每条路径都配了可点按钮：

- 回到当前任务区
- 打开完整路线页
- 直接触发补救图解 / 复测题 / 巩固资源生成

这意味着反馈阶段不再只是“看一段话”，而是能直接进入后续动作。

## 设计取向

这一轮的设计重点不是增加更多信息，而是减少犹豫：

- 用户少想一步
- 页面少解释一层
- 系统多替用户做一次判断

## 价值

这轮优化后，导学主流程更接近真正的学习闭环：

1. 做任务
2. 提交证据
3. 看反馈
4. 直接按建议进入“继续 / 补救 / 复测”
