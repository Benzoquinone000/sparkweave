# 学习画像 P15：把调度结果写成人话

这一轮继续优化的是“完成总结”的可读性。

## 问题

前面已经有了：

- 反馈分流标签
- 学习轨迹里的推进 / 补救 / 复测标签
- 报告里的反馈摘要统计

但如果让普通用户或评委快速理解，单纯看一堆标签还是不够省力。

最自然的需求其实是：

> 你直接告诉我，这一轮我大概是怎么学过来的。

## 本轮目标

把反馈分流结果进一步翻译成自然语言回顾。

## 本轮实现

新增了一段“人话总结”逻辑，会根据最近的反馈分流结果，自动生成类似这样的回顾：

- 这一轮你主要经历了：继续推进 3 次、稳一下 1 次、先补救 1 次。
- 说明你的学习推进整体比较顺畅，系统多数时候可以直接带你进入下一步。

如果当前主要是补救或复测，也会切换成对应语气：

- 更适合先把错因补清楚，再继续推进
- 系统正在更重视“补完之后要重新确认”
- 已经有基础，但仍建议先压实再往前走

## 价值

这一层虽然不复杂，但很关键：

- 用户更容易看懂自己的学习过程
- 评委更容易看懂系统的动态调度逻辑
- 整个“学习效果评估”更像一个真正的复盘总结，而不是数据面板

## 下一步建议

下一步可以继续做两件事中的一件：

1. 把这种“自然语言复盘”同步到画像页，让画像也能总结你的典型推进模式
2. 在课程产出包里补一段“学习策略演化摘要”，方便直接用于展示和答辩
