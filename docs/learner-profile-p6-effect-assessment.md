# 学习画像 P6：学习效果评估融合

## 目标

P6 将学习效果评估从“只看当前导学路线”升级为“当前路线 + 统一学习画像 + 证据账本”。这样系统能回答两个更接近真实教学的问题：

- 这次学习表现如何？
- 这次表现和长期画像里的薄弱点、偏好、校准记录是否一致？

## 当前实现

- `GuideV2Manager.evaluate_session()` 会输出 `learner_profile_context`。
- `build_learning_report()`、`build_study_plan()`、`build_coach_briefing()`、`recommend_resources()` 都会携带同一份画像评估上下文。
- `effect_assessment.dimensions` 新增 `longitudinal_profile` 维度，用于衡量长期画像证据对本次评估的可用性。
- 风险信号会纳入长期薄弱点，例如“长期画像仍显示薄弱点：公式推导”。
- 下一步建议会优先带入统一画像推荐，避免单次 session 结束后忽略长期问题。
- 前端导学报告新增“长期画像依据”卡片，只展示用户能理解的摘要、薄弱点和画像建议，不展示原始 JSON。

## 画像中心里的效果评估

除了导学报告，本阶段还把学习效果评估前移到了统一画像中心 `/memory`，让用户和评委都能更直观看到“系统如何判断学习是否真的在进步”。

前端新增了“学习效果评估”面板，基于统一画像现有数据生成四个维度：

| 维度 | 前端含义 |
| --- | --- |
| `知识掌握` | 当前已进入掌握跟踪的知识点平均状态 |
| `练习表现` | 最近一段时间的题目正确率 |
| `判断稳定性` | 证据条数、来源数、校准次数共同决定的稳定度 |
| `薄弱点压力` | 当前是否仍有明显待补强点 |

系统会把这四个维度再汇总成一个总状态：

- `状态良好`
- `正在进步`
- `仍需补强`

并给出一条更接近教学决策的话术：

- 当前更适合继续推进
- 更适合先做复测
- 更适合补充证据
- 更适合先处理薄弱点

## 画像评估与导学动作的联动

本阶段最重要的补充，不是“多一个评估面板”，而是让评估开始影响导学前端动作：

- 导学页新增“当前导学策略”卡片。
- 导学页会读取统一画像，并结合：
  - 画像可信度
  - 练习正确率
  - 掌握度均值
  - 薄弱点数量
  - 资源偏好
  - 最近一次反馈得分
- 自动判断当前更适合先：
  - 看图解
  - 做练习
  - 看短视频
- 三个资源按钮会自动高亮当前推荐项，形成“评估 -> 动作”的前端闭环。

## 用户可见的闭环提示

为了让评估不是“后台黑盒”，导学页还新增了用户可见反馈：

- 提交当前任务 / 前测 / 练习 / 画像对话后，会主动触发统一画像刷新。
- 导学反馈卡会显示“画像闭环”提示。
- 用户可以直接点击“查看画像变化”，回到画像中心查看前后差异、成长轨迹和效果评估。

## 评分逻辑

学习效果评估目前由六个维度组成：

| 维度 | 含义 |
| --- | --- |
| `progress` | 当前导学路线任务完成进度 |
| `mastery` | 当前路线知识点掌握度 |
| `evidence` | 可评分证据数量和质量 |
| `engagement` | 资源、练习、画像对话等参与行为 |
| `remediation` | 错因补救和复测闭环是否完成 |
| `longitudinal_profile` | 长期画像的证据量、来源数、校准次数和薄弱点可用性 |

其中长期画像不是简单加分项，而是用于解释和调整策略：如果长期画像持续指出某个薄弱点，报告和推荐会提醒先补齐它；如果画像尚未校准，会提示用户到画像中心确认或修正判断。

## 设计原则

1. 当前表现和长期画像分开看：避免一次高分掩盖长期薄弱点，也避免旧画像压过新证据。
2. 只展示结论和依据：前端不展示 JSON，只展示“长期薄弱点”“画像建议”“纳入了多少证据”。
3. 画像可校准：如果画像仍是只读或未校准，评估会提示用户确认关键判断。
4. 推荐可执行：画像信号最终要落到“生成什么资源、做什么练习、补什么任务”。
