PPT 与 7 分钟演示视频建议

本篇说明 SparkWeave 的 PPT 和 7 分钟演示视频可以怎样组织。原则只有一个：按学生完成一次学习的过程讲，不按代码模块讲。

一句话定位

SparkWeave 是面向高校课程的多智能体学习工作台。学生从学习目标出发，系统根据课程资料、学习记录和学习画像生成路径、资源、练习和反馈；资料问答会展示来源，学习结果会回到记录和画像中，影响下一步建议。

这句话适合放在 PPT 第 1 页或视频开头 15 秒内。

答辩主张

PPT 里不要只写“我们用了大模型、多智能体、RAG”。更好讲的是：SparkWeave 把这些技术变成了一条学生能走完的学习路线。

可以直接用下面这组表达：

  • 不是聊天框，而是课程学习闭环：课程目标、资料、资源、练习、记录和下一步建议连在一起。
  • 不是静态画像，而是证据画像：练习、导学任务、资源生成、笔记和学生校准会写进追加式证据账本，再聚合成画像。
  • 不是普通 RAG，而是 Agentic RAG：复杂问题会生成 query plan，多路并发检索后合并来源，用覆盖度和相关性质量门决定是否修复或回退。
  • 不是展示一堆 Agent，而是自动调度：LearningCapabilityRouter 根据任务、约束和画像提示，把请求交给解题、出题、图解、动画、研究或直接工具。
  • 不是只在网页里答疑，而是可延伸到真实学习入口：课程助教支持通道配置，代码层已有 QQ 私聊、群聊和定时提醒发送链路。
  • 不是把讯飞工具放在角落，而是嵌入学习环节：星火模型、Spark Embedding、ONE SEARCH、OCR/公式/图片理解、TTS/ASR/语音评测和星辰工作流分别进入资料、辅导、讲解和评估链路。

PPT 结构

建议控制在 10 到 12 页，页数不必多，重点是每页都能对应真实系统。

7 分钟视频讲稿

视频建议录真实前端，不要只放 PPT。下面时间只是参考，录制时可以略有浮动。

时间 页面 讲法
0:00-0:30 学习 “我们做的是一个面向高校课程的个性化学习工作台。学生进入后先看到学习、资料、记录、设置，而不是一堆陌生配置。”
0:30-1:10 学习 选择 深度学习，展示课程路线和当前任务，说明这是仓库内完整课程模板。
1:10-1:50 记录 / 画像 展示薄弱点、偏好和下一步建议，说明画像来自学习记录和反馈，不是静态标签。
1:50-2:40 资料 打开资料库，展示课程资料、入库状态和资料来源，为后面的问答铺垫。
2:40-3:40 问问题 围绕资料提问，重点展示 Agentic RAG 如何拆分问题、合并来源和判断证据强弱。
3:40-4:35 问问题 / 学习 生成图解、练习、语音讲解或资源建议，说明系统在后台自动调度智能体。
4:35-5:20 课程助教 展示长期课程助教、提醒任务和通道入口，说明可接 QQ 私聊、群聊和定时提醒。
5:20-6:05 学习 完成一次任务或提交反馈，展示任务状态和学习建议变化。
6:05-6:40 设置 展示讯飞相关配置：星火、Embedding、ONE SEARCH、OCR、公式识别、图片理解、TTS、ASR、语音评测、星辰工作流。
6:40-7:00 记录 / 总结 回到记录或学习页，总结“画像 -> 路径 -> Agentic RAG -> 自动调度 -> 评估”的连续体验。

前端录制导览脚本

仓库里提供了一个辅助录屏脚本，作用不是替你生成视频，而是把真实前端按演示顺序打开、停顿、滚动，并在页面左下角放一条讲解提示。录制时你可以开着 OBS 或系统录屏，让脚本带着浏览器走，自己按提示讲。

推荐先启动完整服务：

docker compose up --build

确认 http://127.0.0.1:3782 能访问后，在另一个终端运行：

cd web
npm run record:tour

默认路线会从项目首页开始，再进入学习、资料、问问题、练习、记录、画像、设置、课程助教，最后回到学习页收束。每一步都会停留一段时间，并显示这一页该讲什么。

如果想自己控制节奏，用手动模式：

cd web
npm run record:tour -- --manual

每讲完一页，在终端按 Enter 继续。这个模式最适合正式录制，因为可以避开讲快或讲慢的问题。

常用参数：

命令 适合场景
npm run record:tour 7 分钟视频的默认路线
npm run record:tour -- --manual 正式录制，自己控制翻页
npm run record:tour -- --profile=short 快速检查首页、学习、资料、问问题、画像、设置、助教
npm run record:tour -- --profile=full --step-ms=22000 展示更多页面，并给每页更长讲解时间
npm run record:tour -- --url=http://127.0.0.1:3782 --no-start-server 只使用已经启动的 Docker 前端，不自动拉起 dev server

脚本只做页面跳转、滚动、输入框预填和临时提示，不会替你提交表单、上传文件、删除数据或创建助教。问问题页会预填一个 CNN 讲解问题，正式录制时可以停在那里手动点击发送，展示真实模型返回和来源证据。

讲解重点

PPT 和视频都建议反复强调三件事：

  • 系统有完整课程主线：data/course_templates/deep_learning/deep_learning_foundations.json，并已接入第二门“智能机器人系统”课程。
  • Agentic RAG 的 query plan、子查询结果、质量报告和修复/回退原因会返回到页面，回答有证据链。
  • 多智能体不是菜单，而是由 LearningCapabilityRouter 和默认 chat graph 根据学习任务自动调度。
  • 课程助教可以通过通道配置进入 QQ 等学习入口，支撑长期陪伴和提醒；没有真实凭据时也会明确停留在可演示的 Web/内存通道。
  • 学习行为会写入 evidence.jsonl，画像快照再以精简提示影响下一步任务与资源推荐。

这三件事最能区分 SparkWeave 和普通聊天工具。

讯飞工具如何讲

设置页不要只当配置页讲。可以按学习流程说明：

学习环节 可讲的讯飞能力
课程资料入库 Spark Embedding、OCR
外部资料补充 ONE SEARCH
图片或公式题 公式识别、图片理解
即时辅导 星火大模型、MaaS Coding
多模态讲解 TTS、ASR
学习效果证据 语音评测
课程成果输出 星辰工作流

如果某项服务没有真实密钥，录制或说明时要说清楚它是替代结果，不要说成真实云端调用。

录制前检查

录屏前建议按下面顺序确认：

  • Docker Compose 已启动，http://127.0.0.1:3782 可访问。
  • 学习页能看到 深度学习。
  • 资料页已有可公开展示的课程资料，或准备好现场上传文件。
  • 问问题页能得到带来源的回答。
  • 记录页能看到学习画像、薄弱点或下一步建议。
  • 设置页不会露出完整 API Key。
  • 使用的截图来自 web/screenshots-*.png，并与当前页面一致。

常见失分点

问题 建议
按技术模块逐个讲 改成按学生学习过程讲
只展示问答，没有课程主线 先选课程，再问资料
只说用了大模型，没有证据来源 展示资料来源和引用片段
讲画像但没有行为证据 展示学习记录、反馈或练习结果
讲讯飞工具但没有配置入口 打开设置页和连接检测
视频时间超长 保留主线，少讲边缘功能
展示真实密钥 提前遮罩或切换到安全配置

边界说明

这篇不是论文式说明，而是给展示材料使用的提纲。正式视频和 PPT 应该以可运行系统为准，截图只作为辅助定位。录制时如果使用示例数据或替代结果，应当主动说明，避免被误解为真实学生数据或真实云端服务返回。
