AI Coding 使用说明

本篇用于回应赛题中的“如若使用 AI Coding 工具，给出相关说明”。SparkWeave 在开发过程中使用过 AI 编程助手，但作品的需求取舍、系统设计、密钥配置、真实服务开通、运行验证、视频录制和最终提交由团队负责。

使用范围

AI 编程助手主要参与了这些工作：

范围 具体用途
代码梳理 帮助阅读后端服务、前端页面、资料检索、学习画像和配置相关代码
前端优化 辅助整理学习、资料、记录、设置这条主路径，减少页面噪声
讯飞工具接入 辅助梳理星火、Embedding、ONE SEARCH、OCR、公式识别、图片理解、语音和工作流的配置位置
测试修复 辅助定位 API 合同、课程模板、资料问答和前端构建中的问题
文档整理 辅助把赛题要求、运行方式、数据边界、测试方式和提交材料写成可核验文档

这些工作都需要人工确认。AI 工具给出的建议不会直接作为最终结论，必须能对应真实代码、真实页面、真实配置或真实验证结果。

未交给 AI 工具决定的内容

以下内容由团队自行判断和确认：

  • 赛题理解、作品定位和展示主线。
  • 选择哪一门课程作为主课程。
  • 科大讯飞账号、模型权限和服务开通。
  • .env 中的真实密钥和本地配置。
  • 是否使用真实云端服务或带标记的替代结果。
  • Docker 环境是否可运行。
  • 7 分钟视频录制、PPT 取舍和最终提交包内容。

换句话说，AI 工具参与的是开发辅助，不替代团队对作品真实性和可运行性的责任。

人工复核方式

团队对 AI 辅助内容按以下方式复核：

复核内容 复核方式
文档中的路径 对照仓库真实文件和页面截图
文档中的接口 对照 FastAPI 路由和前端 API 客户端
课程模板 运行 python scripts/check_course_templates.py
API 合同 运行 python scripts/check_web_api_contract.py
项目结构和文档链接 运行 python scripts/check_project_standards.py
前端页面 使用当前前端截图和可运行页面核对
提交边界 检查 .env、真实用户数据和本地运行数据不进入提交包

如果某段文档不能对应项目里的真实文件、真实接口或真实页面，就不作为最终说明。

密钥与隐私边界

AI 编程助手不应接触或生成真实密钥。提交包中不应包含：

  • .env。
  • API Key、API Secret、APIPassword、账号 Token。
  • 本地下载的账号 JSON。
  • 真实学生姓名、联系方式、语音材料、学习记录和个人画像。
  • 未脱敏截图或未授权课程资料。

如果提交材料需要展示学习记录，应使用脱敏样例，只保留课程、知识点、任务、分数和反馈类型。

结果归属

SparkWeave 最终提交的是团队作品。AI 编程助手提供的是阅读、修改、检查和文档辅助；项目是否满足赛题要求，要以可运行系统、源码、课程模板、配置说明、测试结果、演示视频和提交材料为准。

可放入提交材料的简短版本

本项目开发过程中使用 AI 编程助手辅助代码梳理、前端优化、讯飞工具接入、测试修复和文档整理。需求判断、赛题取舍、密钥配置、真实服务开通、运行验证、视频录制和最终提交由团队负责。AI 工具不生成或提交真实密钥；.env、账号凭证、真实学生数据和未脱敏材料不进入提交包。
