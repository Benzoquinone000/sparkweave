# 学习画像 P20：把推进方式压缩成学习协议

这一轮继续围绕“画像要面向使用者”打磨，不增加新配置，也不把更多数据摊给用户。

## 问题

画像页已经能解释推进风格、证据稳定性和最近趋势，但原来的卡片仍然偏“分析面板”：

- 用户需要读多个指标块，才能理解系统准备怎么教。
- 可信度、稳定度、反馈信号容易让页面显得像诊断报表。
- 对普通学习者来说，更重要的是“系统接下来会怎么带我学”。

## 本轮目标

把“推进方式”改成一张更轻的学习协议卡：

1. 用一句话说明系统会按哪种方式带学习者推进。
2. 保留三步策略：起步方式、稳定程度、反馈习惯。
3. 最近趋势只作为一条提示，不抢主任务卡的注意力。
4. 保留进入导学的轻入口，形成“看懂画像 -> 继续学习”的动作闭环。

## 本轮实现

- `/memory` 的“你的学习推进方式”卡片改为更紧凑的三步策略卡。
- 保留“系统接下来会优先”提示，继续说明画像如何影响导学和资源安排。
- 保留 `learner-progress-style-card` 测试锚点，避免前端黑盒主线断裂。

## 价值

- 对用户：少看指标，多看行动。
- 对演示：更容易讲清楚“画像不是表单，而是学习协议”。
- 对后续开发：继续坚持懒人式体验，避免画像页重新变成复杂仪表盘。
