# 学习画像 P11：画像到导学的接力打磨

这一轮不再增加新概念，重点把用户真正会碰到的接力体验做顺。

## 目标

让用户在 `/memory` 看到“下一步建议”后，不需要猜：

- 点进去会发生什么
- 为什么会跳到这个页面
- 到了导学页后先看哪里
- 这一步和后面的前测、资源、反馈是什么关系

## 本轮完成

### 1. 画像页直接说明“点进去后会发生什么”

`NextActionCard` 新增了三步预告：

1. 自动带入目标
2. 自动定位到设置区
3. 创建路线后先做前测

这能明显降低第一次使用导学页时的陌生感。

### 2. 画像页到导学页的落点反馈更明确

前面已经做过：

- `/memory` 跳转到 `/guide`
- 通过 query 携带 `prompt`、`source_label`、`estimated_minutes`
- 额外带上 `target_section=guide-setup-section`
- 导学页自动滚动到设置区并短暂高亮

这一轮把它补成了更完整的“预期管理”链路：用户在点之前就知道会被带到哪里。

### 3. 闭环叙事更顺

现在这条链对用户和评委都更容易讲清楚：

- 画像识别下一步
- 一键进入导学
- 导学自动接住目标
- 前测校准起点
- 资源与练习围绕目标生成
- 反馈回写画像

## 价值

这一轮提升的不是“功能数量”，而是“使用阻力”：

- 用户更敢点“进入导学”
- 用户更少在页面里迷路
- 演示时更容易说清楚接力逻辑
