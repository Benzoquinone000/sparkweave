# 学习画像 P19：让课程产出包解释画像如何驱动成果

P18 已经把推进风格放到画像页。P19 继续把这层信息带到课程产出包里。

## 问题

比赛演示时，评委不一定会跟着完整操作过程走一遍。如果最终产出包只展示项目、Rubric、作品集，仍然需要讲解者额外解释：

> 这些产物为什么是个性化的？

## 本轮目标

让课程产出包自己回答这个问题：画像如何影响课程路线、资源顺序、练习安排和最终作品组织。

## 本轮实现

后端 `GuideV2Manager.build_course_package` 新增 `learning_style`：

- `label`：学习推进风格，例如概念澄清型、练习驱动型。
- `summary`：为什么产出包会这样组织。
- `trend`：最近证据是否稳定、是否可提速、是否仍需补证据。
- `signals`：偏好、卡点、任务/资源/练习证据。
- `demo_talking_point`：可直接用于录屏讲解的一句话。
- `path_effect`：说明画像如何影响路径与资源。

前端课程产出包新增“画像驱动产出”卡片，并把风格标签同步到“录屏检查”区域。

Markdown 导出也新增“学习画像与推进方式”章节，便于比赛文档、PPT 和演示视频复用。

## 价值

- 让最终成果不只是“课程包”，而是“由画像驱动生成的课程包”。
- 降低演示解释成本，评委打开产出包就能看到个性化逻辑。
- 把赛题里的画像、路径规划、资源生成、效果评估串到一个可提交材料里。
