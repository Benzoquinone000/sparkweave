# 学习画像 P8 一步行动建议

P8 的目标是让学习画像不只“展示状态”，还要直接告诉学习者现在最该做的一件事。用户不应该先读完可信度、薄弱点、掌握度和证据来源，再自己推断下一步。

## 设计原则

- 首屏只给一个主行动：补基、练习、继续路线或校准画像。
- 主行动必须能追溯到画像证据，例如薄弱点、掌握度、目标或证据不足。
- 行动文案面向学习者，不展示原始 JSON。
- 行动入口默认跳到导学页，由导学继续完成资源生成、练习、反馈和路径调整。

## 后端落地

`LearnerProfileSnapshot` 新增 `next_action`：

```json
{
  "kind": "remediate",
  "title": "先补齐「公式推导」",
  "summary": "已有多条证据指向这个薄弱点，建议先用一个短任务把它稳住。",
  "primary_label": "进入导学补基",
  "href": "/guide",
  "estimated_minutes": 15,
  "source_type": "weak_point",
  "source_label": "公式推导",
  "confidence": 0.78,
  "suggested_prompt": "围绕「公式推导」安排一个 15 分钟补基任务，先生成图解，再用 3 道小题确认理解。"
}
```

优先级：

1. 有薄弱点：生成补基行动。
2. 没有薄弱点但掌握度证据不足：生成练习行动。
3. 有明确目标且状态稳定：继续当前导学路线。
4. 证据不足：先校准画像。

导学创建接口支持可选 `source_action`。当用户从画像行动卡进入导学时，前端会把行动来源、类型、关联薄弱点、可信度和建议 prompt 一并提交，后端写入 `course_map.metadata.source_action` 和首个任务 `metadata.source_action`。

## 前端落地

`/memory` 首屏新增“现在只做这一步”行动卡：

- 显示行动标题、摘要、预计时长和依据可信度。
- 展示可直接对导学说的提示词。
- 提供进入导学的主按钮，并把 `suggested_prompt` 与行动来源通过 `/guide?prompt=...&new=1` 带入新的导学路线输入框。

这使画像中心从“状态看板”变成“学习导航”，更符合懒人式、递进式的导学体验。

## 后续

- 行动完成后写回 evidence，形成“建议 -> 行动 -> 反馈 -> 画像更新”的闭环。
- 后续可以将 `next_action.kind` 映射到不同导学模板，例如补基模板、复测模板、迁移模板。
