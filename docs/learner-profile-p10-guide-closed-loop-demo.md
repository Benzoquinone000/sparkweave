# 学习画像 P10：导学闭环总览卡

## 目标

把“学习画像 -> 导学任务 -> 资源生成 -> 反馈回写”的链路压缩成一个用户和评委都能一眼看懂的前端总览卡，减少解释成本。

## 落地位置

- 页面：`/guide`
- 位置：导学页顶部进度区下方、主内容区上方

## 当前实现

新增“学习闭环总览”卡片，分成四步展示：

1. 画像输入
2. 导学拆解
3. 资源生成
4. 反馈回写

每一步都同时展示：

- 一句人话摘要
- 2 个关键指标
- 当前状态标签

## 卡片内容来源

### 1. 画像输入

来自统一学习画像：

- `overview.current_focus`
- `learning_state.weak_points`
- `confidence`

### 2. 导学拆解

来自当前导学会话：

- `current_task`
- `session.tasks`
- `report.overview.completed_tasks`
- `report.overview.total_tasks`

### 3. 资源生成

来自当前任务的资源产物：

- `currentTask.artifact_refs`
- 当前资源类型汇总

### 4. 反馈回写

来自最近一次学习反馈与时间线：

- `learning_feedback.score_percent`
- `learning_timeline.recent_events`
- `learning_report.overview.progress`

## 用户价值

- 用户能快速看懂系统现在在做什么
- 用户能理解“为什么推荐这个资源”
- 用户能看到自己的操作如何回写到画像

## 演示价值

这张卡很适合比赛答辩或录屏时直接讲：

> 系统先根据学习画像识别重点和卡点，再把目标拆成当前任务，按画像生成资源；用户提交练习和反思后，反馈会回写画像，并继续调整下一步。

## 后续可继续增强

1. 加入小型动画连线，让四步之间的关系更直观
2. 支持点击某一步后滚动到对应详细模块
