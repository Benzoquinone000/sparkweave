# 前端设计说明

SparkWeave 的前端不是把所有智能体、RAG、诊断、工具都摆成菜单，而是先让学生知道下一步该点哪里。页面默认围绕四个入口展开：学习、资料、记录、设置。其他能力放进“更多工具”，需要时再打开。

这不是弱化技术，而是把技术放回学习过程里：学生先学习、查资料、看记录、改设置；系统在后台完成多智能体分工、资料检索、画像更新和资源生成。

## 主入口为什么这样安排

| 一级入口 | 学生要完成的事 | 背后的能力 |
| --- | --- | --- |
| 学习 | 继续课程、创建导学路线、完成练习和反馈 | 个性化路径、课程模板、学习效果评估 |
| 资料 | 上传课程资料、问资料、查看引用来源 | RAG、OCR、Embedding、资料质量判断 |
| 记录 | 查看下一步建议、学习证据和画像修正 | 学习画像、证据记录、下一步动作 |
| 设置 | 连接模型、搜索、OCR、语音和讯飞工具 | 服务配置、连接检测、密钥遮罩 |

问答、练习、课程助教、图像解题等工具仍然存在，但它们被放在学习工具和高级工具里，避免新用户一进来就面对太多陌生选项。

## 页面文案的取舍

前端文案尽量用学习动作，而不是技术名词。

| 页面 | 典型文案 | 想解决的问题 |
| --- | --- | --- |
| 学习页 | “一步一步来” | 学生知道从课程路线开始 |
| 资料页 | “上传资料，然后直接提问” | 学生知道先放资料，再问资料 |
| 记录页 | “先看系统建议你做什么” | 学生知道画像不是抽象标签，而是下一步建议 |
| 设置页 | “需要改什么，就进对应页面” | 学生知道配置项被分成几类 |

这类文案不是装饰，它直接决定评委或学生第一次打开系统时能不能快速完成一次学习流程。

## 空状态和下一步

前端空状态必须给出下一步动作，避免只显示“暂无数据”。

| 场景 | 页面给出的下一步 |
| --- | --- |
| 没有资料库 | 引导创建资料库并上传课件、论文或笔记 |
| 资料库没有文档 | 引导上传资料，处理完成后再提问 |
| 学习记录不足 | 引导先完成导学或练习，让系统建立画像 |
| 配置不可用 | 引导进入连接检测或重新保存服务配置 |
| 来源不足 | 引导补资料、换问法或做资料库预检 |

这样处理后，系统不会把学生丢给配置项或技术日志，而是始终给出一个可以继续的动作。

## 后台能力如何收起来

赛题要求多智能体、RAG、学习画像、多模态生成和效果评估，但这些不一定都适合成为一级菜单。SparkWeave 的处理方式是：

| 后台能力 | 前端呈现方式 |
| --- | --- |
| 多智能体协作 | 在问答过程里显示“查资料、生成练习、生成图解”等状态 |
| RAG 证据链 | 展开“资料来源”查看来源片段和检索过程 |
| 学习画像 | 在记录页用当前重点、卡点和下一步行动表达 |
| 学习效果评估 | 在学习反馈和记录页体现掌握度、错因和下一步建议 |
| 讯飞工具 | 在设置页配置，在问答、资料、图像和语音流程里使用 |
| 高级能力 | 放入更多工具，不打断学习主线 |

如果要核验技术细节，可以进入对应页面展开；普通学生则可以只沿着学习主线走。

## 典型核验路径

前端核验建议按学生学习过程组织，而不是按代码模块组织：

1. 进入学习页，选择一门完整课程，创建导学路线。
2. 完成一次前测或练习，让系统生成反馈。
3. 进入资料页，查看课程资料和资料问答来源。
4. 进入问问题页，提出一个课程问题，展示图解、练习或语音讲解。
5. 回到记录页，看画像、薄弱点和下一步建议是否变化。
6. 进入设置页，展示讯飞模型、Embedding、搜索、OCR、语音和连接检测。

这条路径能把赛题核心要求串起来：画像、路径、资源、辅导、评估和讯飞工具都在同一次学习流程中出现。

## 移动端与窄屏

前端同时考虑桌面和移动端：

| 设计点 | 说明 |
| --- | --- |
| 移动底部导航 | 保留学习、资料、记录、设置等高频入口 |
| 抽屉导航 | 更多工具在移动端用抽屉展开 |
| 卡片宽度 | 关键卡片在窄屏下纵向排列，避免文本挤压 |
| 状态提示 | 运行状态和动态面板保留，但不抢主流程 |

移动端截图包括 [移动学习页](../../web/screenshots-mobile-guide.png)、[移动资料页](../../web/screenshots-mobile-knowledge.png) 和 [移动问问题页](../../web/screenshots-mobile-chat.png)。

## 边界说明

- 前端优先服务真实学习路径，不把每个后端能力都做成一级入口。
- 截图只能作为定位材料，最终以可运行系统为准。
- 设置页会遮罩密钥，录屏时仍应注意不要展示真实完整凭证。
- 如果新增功能，默认先判断它服务学习、资料、记录还是设置；只有确实高频，才考虑放到主入口。
