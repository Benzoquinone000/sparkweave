# 学习画像 P14：让动态调度变得可回看

这一轮继续补的是“反馈之后的可解释性”。

## 问题

前一轮已经把反馈页改成了更清楚的分流：

- 继续推进
- 先补救
- 再复测

但如果用户回头看整条学习路径，仍然不一定能一眼看出：

> 我这一轮到底经历了几次补救、几次推进、几次复测？

## 本轮目标

把“动态调度”从即时反馈卡扩展到：

- 学习轨迹
- 学习效果报告

也就是不仅当下能用，回头复盘时也能看懂。

## 本轮实现

### 1. 学习轨迹增加调度标签

在时间线事件里，如果事件带有学习反馈，会进一步提炼出短标签：

- 继续推进
- 稳一下
- 先补救
- 去复测
- 做巩固

这样用户在时间线里扫一眼，就能看出这一段学习是“顺畅推进”还是“补救后再推进”。

### 2. 学习轨迹顶部增加分流汇总

最近几条学习轨迹会再汇总成一组轻量标签，例如：

- 继续推进 ×2
- 先补救 ×1
- 去复测 ×1

不做重图表，只做轻提示，避免页面重新变杂。

### 3. 学习效果报告增加分流回顾

在“反馈摘要”里，现在不仅能看：

- 反馈次数
- 稳定次数
- 需干预次数

还会额外显示这一轮的调度分布，比如：

- 继续推进 ×3
- 稳一下 ×1
- 先补救 ×1

同时最近一次反馈也会带上对应短标签。

## 价值

这一轮补上的，是比赛里很重要的一层“学习效果评估可解释性”：

- 用户能看懂系统不是随机改路线
- 评委能看懂系统确实在持续调整学习策略
- 导学闭环从“即时可用”变成“过程可回看”

下一步可以继续做两件事中的一件：

1. 在完成总结里补一段自然语言回顾，例如“你这一轮经历了 1 次补救、1 次复测、3 次顺畅推进”
2. 把这些动态调度结果进一步回写进学习画像，让画像页也能直接展示“你的典型学习推进模式”
